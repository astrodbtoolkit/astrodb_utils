# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

`astrodb_utils` provides functions to build, query, and ingest data into astronomy databases that follow the [astrodb-template-db](https://github.com/astrodbtoolkit/astrodb-template-db) Felis schema. It is a utility layer on top of `astrodbkit` (SQLAlchemy-based). Only SQLite databases are supported.

## Setup and commands

```bash
pip install -e ".[test]"    # dev install (Python >= 3.11); use .[all] to also get docs deps

# REQUIRED before running tests: clone the template database into tests/
git clone https://github.com/astrodbtoolkit/astrodb-template-db.git tests/astrodb-template-db

pytest                                            # run all tests (run from repo root — conftest uses relative paths)
pytest tests/test_sources.py::test_ingest_source  # run a single test
pytest -o log_cli=1 --log-cli-level=DEBUG         # with live log output

ruff check .        # lint (line length 120); CI runs lint non-blocking (--exit-zero)
ruff format .       # format
pre-commit install  # installs ruff + ruff-format hooks

sphinx-build docs docs/_build/html   # build docs (needs .[docs] extras + pandoc installed)
```

- Tests that hit NASA ADS are skipped unless the `ADS_TOKEN` environment variable is set.
- Many tests call live services (SIMBAD via astroquery), so they need an internet connection.
- `astrodb_utils/version.py` is generated by setuptools_scm — never edit it.
- CI (`.github/workflows/run_tests.yml`) tests Python 3.11–3.13 and checks out the template db into `tests/astrodb-template-db` the same way.

## Architecture

### The AstroDB data model

A database "lives" as a directory of JSON files (the `data/` path) plus a Felis `schema.yaml` and a `database.toml` settings file — the JSON files are the source of truth. The `.sqlite` file is a disposable build artifact:

- `build_db_from_json()` (`loaders.py`) deletes any existing `.sqlite` file and rebuilds it from schema + JSON. Settings come from `database.toml` (parsed by `DatabaseSettings`), with keyword arguments as overrides. Also exposed as the `build_db_from_json` console script.
- `read_db_from_file()` connects to an existing `.sqlite` file.
- `load_astrodb()` (`utils.py`) is deprecated in favor of the two functions above.

The schema distinguishes ordinary tables from lookup/reference tables (Publications, Telescopes, Instruments, Versions, PhotometryFilters, Regimes, AssociationList, ParameterList, CompanionList, SourceTypeList). The default list lives in `loaders.DatabaseSettings` and can be overridden via `lookup_tables` in `database.toml`.

### Ingest modules and shared conventions

`sources.py`, `publications.py`, `photometry.py`, `spectra.py`, and `instruments.py` each provide `ingest_*` functions plus `find_*`/`get_db_*` lookup helpers. Shared conventions:

- First argument is always the `astrodbkit.astrodb.Database` object.
- Nearly every public function takes `raise_error: bool = True`. Failures route through `exit_function(msg, raise_error)` in `utils.py`, which either raises `AstroDBError` or logs a warning and returns None — preserve this pattern in new ingest code.
- Ingest functions validate against lookup tables before inserting (e.g., `ingest_source` requires its reference to already exist in Publications; photometry resolves the filter, spectra resolve instrument/mode and regime first). Inserts use plain SQLAlchemy: `with db.engine.connect() as conn: conn.execute(db.<Table>.insert().values(data)); conn.commit()`, catching `sqlalchemy.exc.IntegrityError`.
- Lookups are case-insensitive (`ilike`) and tolerant: `find_source_in_db` cascades exact name → fuzzy → SIMBAD resolution → coordinate region search; `find_publication` falls back to name-fragment matching; `get_db_regime` retries with hyphens stripped.

### External services

- **SIMBAD** (astroquery): name/coordinate resolution in `sources.py`; guarded by `use_simbad` flags and `internet_connection()`.
- **NASA ADS**: `publications.py` fetches bibcode/DOI/description metadata; requires `ADS_TOKEN` env var (`check_ads_token()`); pass `ignore_ads=True` to skip.
- **SVO Filter Profile Service**: `fetch_svo()` in `photometry.py` retrieves filter wavelengths for `ingest_photometry_filter`.
- `fits.py` is service-light: helpers to make FITS headers comply with the IVOA spectrum data model (`ivoa-spectrum-dm-1.2`) before spectra ingestion; `spectra.py` verifies spectra are loadable/plottable with specutils.

### Logging

`__init__.py` configures the parent `astrodb_utils` logger (INFO, stdout); modules use `logging.getLogger(__name__)`. Some functions branch on `logger.parent.level <= 10`, so the parent log level changes behavior (e.g., verbose database loading), not just output volume.

### Tests

`tests/conftest.py` defines a session-scoped, autouse `db` fixture that builds a fresh database (`tests/astrodb-template-tests.sqlite`) from the cloned template repo and seeds two publications (`Refr20`, `Prob83`) referenced throughout the tests. All tests share that one database instance, so ingest tests mutate shared state and some use `pytest-dependency` for ordering. Pytest is configured (`norecursedirs`) not to collect tests from the cloned `astrodb-template-db` directory.
